# Scope

This document holds stable constitutional rules — specific, operative
constraints governing the investigation or the ecosystem model — as
distinct from 17_PHILOSOPHY.md, which holds the underlying worldview
those rules follow from. Several candidate rules were excluded here
specifically because they turned out to be applications of a principle
already documented there (noted inline where relevant). Product
architecture, features, workflows, and design proposals are out of
scope even where a related abstract principle is in scope — noted
explicitly where this distinction applies.

# Rules Governing the Wall (Evidence Base)

**Every UDE belongs to exactly one stakeholder.** A UDE always
represents one stakeholder's lived reality.
*Origin: stakeholder-first Wall reorganization — stated as "a strict
methodological rule."*

**Never merge overlapping UDEs across stakeholders.** If two
stakeholders appear to experience the same problem, both observations
are preserved separately, in each stakeholder's own language — never
collapsed into one generic statement, even where they clearly describe
the same interaction from opposite sides.
*Origin: same reorganization — stated as "a strict methodological
rule."*

**The stakeholder list is fixed at the start of the investigation; the
UDE count is not.** A stakeholder identified as part of the ecosystem
remains listed at zero observations until real investigation changes
that count — a stakeholder is never removed from view for lack of
data.
*Origin: standing rule established during the Wall's stakeholder-first
reorganization.*

**A UDE is never deleted.** Retirement requires an individually stated
reason and remains visible in the record; blanket or category-level
retirement without an individual reason is not permitted.
*Origin: applied consistently during the Ecosystem/Meta/Data &
Intelligence retirement passes.*

**Every observation carries two evidence classifications:** a
provenance rating (Documented / Observable / Inferred / Candidate) and
an audit status (Verified / Partially Verified / Pending / Not
Audited).
*Origin: the Wall's evidence model, applied to every UDE catalogued.*

# Rules Governing CRT and Cloud Construction

**Objectives and Needs are elicited, not proposed.** They belong to
participants and already exist prior to investigation; they are not
invented or assigned by the investigator.
*Origin: foundational methodology document.*

**A Cloud is not a node.** It is a diagnosed relationship — the
incompatibility between two Needs under current Assumptions — not an
entity with the same standing as a Structural Reality or a UDE. Where
a Cloud is drawn as a shape in a diagram, this is a representational
convenience only.
*Origin: foundational methodology document, restated explicitly during
Ecosystem Cloud construction.*

**Do not force a Cloud, but expect one.** Causality is followed
without manufacturing a conflict that isn't there. At the same time, a
UDE is expected to resolve into a Cloud somewhere along its causal
chain, because a chronic, unaddressed effect implies a blocked
resolution — an unexplained termination that reaches neither a Cloud
nor a Structural Reality signals an incomplete causal chain, not
equilibrium.
*Origin: initial rule stated during stakeholder-first CRT construction
("do not search for Clouds... follow causality"); refined later to the
current form after direct discussion of why most UDEs should be
expected to resolve into a conflict rather than terminate
unexplained.*

**Every CRT-level node must trace back to the specific Wall
statement(s) it synthesizes.** Many-to-one synthesis is expected and
normal; the mapping must remain visible and auditable rather than
implied.
*Origin: the "translation layer" established between the Wall's
first-person language and the CRT's systems-level language.*

**Investigation proceeds in a fixed sequence:** Minimal Stakeholder
UDEs → Minimal Stakeholder CRTs → Minimal Ecosystem CRT → Internal
Clouds → Intra-Stakeholder Clouds → Inter-Stakeholder Clouds →
Injections → Working Future Reality Tree → Predicted Emergent
Behaviours → Product Architecture → Implementation → Observed
Ecosystem Behaviour → Refinement. Each stage is investigated and, where
applicable, frozen before the next begins.
*Origin: agreed investigation pipeline, with "Minimal Stakeholder CRTs
→ Minimal Ecosystem CRT" specifically replacing an earlier
"Minimal Working CRT" formulation per stakeholder.*

**Every entry in 13_OBJECTIVES.md and 14_NEEDS.md must state its Level
explicitly** — Stakeholder or Ecosystem — rather than relying on file
position or heading alone.
*Origin: stated as a mandatory field requirement in the frozen
documentation structure.*

# Rules Governing the Documentation Repository

**Every artifact defined in the frozen Documentation Structure appears
exactly once in 02_ROADMAP.md — no more, no less.** If a new artifact
is needed, it is added to the structure first, and only then receives
a roadmap entry.
*Origin: standing rule stated in 02_ROADMAP.md.*

**Every file in KNOWLEDGE/ and THINKING_PROCESSES/ carries a Status,
Version, Last Updated, and Depends On header.** 00_MANIFEST.md is
derived from these headers; it is not independently authored, so that
the manifest and file content cannot drift apart.
*Origin: stated in the frozen documentation structure.*

# Rules Governing the Ecosystem Model

The following were recorded as Stable Principles prior to this file's
creation. They are transcribed here verbatim; no elaboration has been
established for any of them beyond what is stated.

- Opportunity is operationally primary.
- Fit is preferred over exhaustive search.
- Evidence should be generated as close to its source as possible.
- Opportunities decay.
- Renewal is evidence renewal.
- Constitutional neutrality is maintained.
- Brokerage relationships remain broker-centric.
- Discovery is distinct from trust.

*Origin: Investigation Status record, "Stable Principles."*

**Information quality is judged by four standing preferences:** signal
over volume; evidence over persistence; freshness over permanence;
propagation over duplication.
*Origin: Investigation Status record, "Information Quality."*

# Explicitly Excluded (Product/Design Scope, Not Repeated Here)

"Constitutional neutrality is maintained" (above) is an ecosystem-model
principle and is in scope. The specific architectural mechanisms
discussed at one point for achieving it (e.g. a proposed matching
network having no consumer-facing surface, no pay-for-visibility
revenue, no in-house brokerage) are product/design proposals and are
out of scope for this document — they belong in PRODUCT/CONSTITUTION.md
(not yet drafted) if adopted there.
